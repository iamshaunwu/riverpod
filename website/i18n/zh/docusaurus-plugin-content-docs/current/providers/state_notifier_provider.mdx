`StateNotifierProvider` 是个用来监听和提供 [StateNotifier]（来自Riverpod 重新导出的 [state_notifier] 包）。它与 [StateNotifier] 结合一起并使用于管理用户交互而发生变化的状态是 Riverpod 推荐的解决方案。

它通常用于：

- 提供一个在响应自定义事件后而随时间变化的 **不可变** 的状态。
- 中心化逻辑以修改某些状态（即“业务逻辑”）的单个位置，从而提高了随着时间的推移的可维护性。
